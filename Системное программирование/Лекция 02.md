## Очередь сообщений System V (msg q)

"Это обычная классическая очередь"

В ядре существует таблица очередей сообщений, где каждой  очереди соответсвует некоторая запись.

Атрибуты очереди сообщений:
- имя (целое число)
- UID, GID (group id) создателя очереди
- UID, GID назначенного владельца
- права доступа к очереди сообщений *(rvx биты??)*
-  время и идентификатор процесса, который последним передал сообщение в очередь
- время и идентификатор процесса, который последний прочитал сообщение из очереди
- указатель на сообщение (список сообщений)

Сообщение состоит из:
- данные сообщения
- тип сообщения

Системные вызовы:
- msg get
- msg send
- msg rev
- msg ctl

Создание/получение (открытие) доступа очереди сообщений
```C++
int msgget(key_t key, int flag)
key - идентификатор (имя) очереди сообщений
flag - флаги

flag:
- 0 получение доступа к сущесствующему сообщению
  
```
IPC_CREAT, IPC_EXCZ
IPC_CREAT | IPC_EXCL - параметризированние создание новой очереди сообщений
IPC_CREAT | 0777 - задание прав доступа

Отправка (запись) сообщения в очередь
```C++
int msgsnd(int msgfd, const void* msgPtr, int len, int flag)
msgfd - целое число, которое получено от msgget
msgPtr -указатель на переменную типа struct msgbuf
len - длина в байтах поля mtext, на который указывает msgPtr

struct msgbuf {
   long mtype;          //Mesage type
   char mtext[MSGMAX];       //Message text
}

flag - флаг
IPC_NOWAIT - вызов будет неблогирующим
Если передать 0, то вызов будет блокирующим (по умолчанию)

```

Получение (чтение) сообщения очереди

```C++
int msgrcv(int msgfd, const void* mmsgPtr, int mtype, int flag)
msgfd - целое число, которое получено от msgget
msgPtr - указатель на переменную, тип которой похож на msgbuf (куда будет считано сообщение)
len - размер сообщение, которое будет считано
mtype - тип сообщения
flag - флаг

Смысл mtype:
0 - счтать из очереди самое старое сообщение любого типа
положительное число - считать из очереди сообщение звдвнного типа
отрицательное число - считать из очереди сообщение, тип которого меньше абсолютного значения mtype или равен ему. Если таких сообщений несколько, то считать самое старое 
```

Управление очередью сообщений

```C++
int msgctl( int msgfd, int smd, struct msgID.dc* msgbufPtr)
msgfd - целое число, которое получено от msgget
cmd - команда
msgID.dc* msgbufPtr - указатель на структуру с полями, которые описывают очередь

Команды:
IPC_STAT - получение атрибутов очереди
IPC_SET - установление атрибутов очереди
IPC_RMID - удаление

msgctl(msgfd, IPC_RMID, NULL) - удаление очереди сообщений

```

```C++

	int val; // значение семафора
	struct ds* buf; // упрравляющие параметры

```
IPC_STAT - получение информации о наборе семафоров
IPC_SET  - модификация управляющих параметров
IPC_RMID - удалить набор семафоров
GETALL - получить текущие значения всех семафоров в наборе
SETALL - установить все значения семафоров с помощью arg array
GETVAL - получить значение семафора с номером num (arg не используется)
SETVAL - установить семафор со значением num в значение arg.val
GETNCNT - скажет, кол-во заблокированных процессов, которые ждут увеличение семафора с номером num
GETZCNT - количество заблокированных процессов, ожидающих превращение семафора в 0

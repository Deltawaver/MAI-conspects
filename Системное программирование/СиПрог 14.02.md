# Синхронизация доступа к разделяемому ресурсу

## Набор из двух семафоров
1) Серверный процесс создает разделяемой ресурс и набор из 2-х семафоров и устанавливает их в [ 0 | 1 ]
2) Серверный процесс ожидает данных клиента. Выполняется попытка открытия семафора
	   semop(-1, 0)
3) Клиенты хотят получить доступ к разделяемому ресурсу и они все вместе хотят открыть набор семафоров [ 0 | 0 ]
	   semop(0, -1)
4) Получивший доступ клиент пишет запрос в разделяемый ресурс [ 1 | 0 ]   [ 0 | 0 ]
	   semop(1, 0)
5) Клиент, который разблокировал сервер, выполняет [ 1 |  1 ]
	   semop(-1, -1)
6) Процесс сервер получает доступ к разделяемому ресурсу и обрабатывает запрос клиента [ 0 | 0 ]
	   semop(1, 1)
   Серверный процесс переходит в режим ожидания, выполняя semop(-1, 0)
7) Клиент, который ожидает ответа от серверного процесса разблокируется, прочитает ответ сервера
   И клиент выполняет semop(0, 1)
   Клиент "погибает" [ 0 | 1 ]

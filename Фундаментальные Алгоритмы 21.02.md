# ООП: Основные парадигмы и реализация в C#

## 1. Три кита ООП

### Инкапсуляция
*   **Определение:** Средство объединения данных и методов для работы с ними.
*   **Суть:** На уровне языка мы можем связать то, что обрабатываем (данные), и то, с помощью чего работаем (методы).

### Наследование
*   **Определение:** Механизм повторного использования кода.
*   **Суть:** Позволяет развивать (расширять или изменять) существующий тип данных, создавая новые классы на основе старых.

### Полиморфизм
*   **Определение:** Способность организовывать единообразную работу с объектами разных типов, имеющих общего предка.
*   **Суть:** "Один интерфейс — множество реализаций".
*   **Виды полиморфизма:**
    1.  **Статический (раннее связывание):** Реализуется через **перегрузку методов** (несколько методов с одним именем, но разными параметрами).
    2.  **Динамический (позднее связывание):** Полиморфное поведение организуется через наследование и **виртуальные функции**.

#### Виртуальные функции
*   Механизм, позволяющий вызвать метод производного класса через ссылку или указатель на базовый класс.
*   Указатель на виртуальную функцию хранится в специальной **виртуальной таблице (vtable)**.

## 3. Особенности языка C# (в контексте консольных приложений)

### Общая архитектура
*   **Обязательная парадигма:** В C# всё строится на классах. Сущности не могут жить вне класса.
*   **Точка входа:** Метод `Main` (с большой буквы), а не `main`.
*   **Статика:** Метод `Main` объявляется статическим (`static`). Статические члены принадлежат не экземпляру, а самому типу.

### Управление памятью (Garbage Collector)
*   Нет деструкторов в классическом смысле C++. Проблемы ручного управления памятью автоматизированы сборщиком мусора.
*   **Типы значений (Value types):** Хранятся в стеке.
    *   *Например:* простые числовые типы, структуры (`struct`).
*   **Ссылочные типы (Reference types):** Память выделяется в куче (heap). Переменная хранит не сам объект, а **ссылку** на него.
    *   *Например:* все **классы**, **массивы**, **строки (`string`)**.

### Работа со строками
*   **`string` (статические строки):**
    *   Строки этого класса неизменяемы. При любой модификации создается новый объект в куче.
    *   Для простых задач подходит, но для обилия операций с модификацией лучше использовать аналог.
*   **`StringBuilder` (динамические строки):**
    *   Рекомендуется для задач, связанных с частым изменением (конкатенацией) строк.

## 4. Простейшая программа на C#

```C#
using System; // Подключаем пространство имен для работы с базовыми классами (например, Console)

namespace mySpace // Собственное пространство имен для организации кода
{
    class Program // Объявление класса
    {
        // Точка входа в программу. Ожидает массив строк (аргументы командной строки).
        // Стандартно Main ничего не возвращает (void).
        static void Main(string[] args)
        {
            // Вывод строки в консоль
            Console.WriteLine("Goodbye World");
        }
    }
}
```
